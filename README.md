기술면접
===========

## 두서 없이 적을거임.

###  객체 지향/ 절차지향의 차이
#### 객체지향

* 프로그램을 객체 단위로 나누어 작성하는 방식. 데이터와 그를 처리하는 메서드를 하나의 단위로 묶어서 생각
* 특징: 캡슐화, 상속 , 다형성<br>
* ex) JAVA , C++ , Python

#### 절차지향
* 프로그램을 프로시저나 함수 등의 절차로 나누어 작성하는 방식으로 기능 중심으로 구현한다.<br>
* 특징: 함수 중심 , 데이터와 코드 분리 , 변수의 전역화 <br>
*  ex) Fortran , C , COBOL


<hr>

### 컴파일러와 인터프리터의 차이 

- 두가지 모두 프로그래밍 언어를 기계어로 변환 하여 실행할때 사용되는 도구
#### 컴파일러

* 소스 코드 전체를 한 번에 컴파일 하여 기계어로 번역하는 방식<br>
- 실행 시간 오버헤드 없음 / 실행 파일 생성<br>
 ex) C, C++, Java (Java는 중간 언어로 변환 후 실행)

#### 인터 프리터

* 소스 코드를 한 줄씩 읽어서 실행하면서 동시에 번역하여 기계어로 실행<br>
* 실행 시간 오버헤드 존재 / 실행 파일 생성 없음<br>
* ex) Python, JavaScript, Ruby

<hr>

### 오버로딩과 오버라이딩의 차이

#### 오버로딩
* 같은 이름의 메서드나 함수를 여러 개 정의하는 것으로, 매개변수의 개수나 타입이 달라야 한다.

#### 오버라이딩
* 부모 클래스에서 정의된 메서드를 자식 클래스에서 재정의하는 것으로, 런타임에 객체의 실제 타입에 따라 동적으로 결정됩니다.

<hr>

### 프레임워크와 라이브러리 차이

#### 프레임 워크

  * 흐름 : 프레임워크가 코드의 흐름을 제어하며 개발자는 프레임워크가 제공하는 틀 안에서 코드를 작성.
  * 사용법 : 프레임워크가 개발자의 코드를 호출하여 동작.
  * 제어 : 프레임워크가 제어하며 개발자는 프레임워크에 따라 동작하도록 코드를 작성.

#### 라이브러리
* 흐름 : 개발자가 코드의 흐름을 제어하며 필요한 기능을 호출하여 사용.
* 사용법 : 필요할 때 호출하여 사용.
* 제어 : 개발자가 제어

### 애자일 방법론이란?

#### 애자일 방법론

* 정의 : 소프트웨어 개발 및 프로젝트 관리에 적용되는 반복적이고 적응적인 접근 방식
* 목표 : 빠르게 변화하는 비즈니스 환경에 대응하면서도 효율적으로 소프트웨어를 개발
* 주요 원칙 : 개인과 상호작용 / 문서보다는 작동하는 소프트웨어를 우선 /고객과의 협력 / 계획 보다는 변화에 유연하게 대처
* 장점 : 고객만족 / 변화 대응 /효율적인 팀 작업 / 제품 품질 향산

<hr>

### 디자인 패턴이란?

#### 디자인 패턴

* 정의 : 소프트웨어 디자인에서 특정한 문제를 해결하기 위해 반복적으로 사용되는 최적의 해결책을 형성한 설계 템플릿
* 유형 : 생성 패턴 / 구조 패턴 / 행위 패턴


<hr>

### MVC란?

#### MVC

* 소프트웨어 디자인 패턴 Model-View-Controller의 약어 
* 모델 : 어플리케이션의 데이터와 비지니스 로직
* 뷰 : 사용자에게 데이터를 시각적으로 표현 하는 부분
* 컨트롤러 : 사용자의 입력을 처리하고 그에 따라 모델을 업데이트하거나 뷰를 업데이트 하는 부분
* 장점 : 코드의 모듈화 / 유지 보수성 향상 / 재사용성 / 확장성 향상

<hr>

### JVM의 구조란?

#### JVM

* 자바 언어로 작성된 프로그램을 실행하기 위한 가상 머신
* 구성 : 클래스 로더 /메모리 영역 / 실행 엔진 / 네이티브 인터페이스 / 네이티브 메소드 라이브러리
